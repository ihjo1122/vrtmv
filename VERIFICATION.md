# VRTMV 검증 가이드

## 사전 요구사항
- Android Studio Ladybug 이상
- Galaxy S23 Ultra (또는 minSDK 31 이상 기기)
- USB 디버깅 활성화

---

## 1단계 검증: 카메라 + AR 오버레이

### 1.1 빌드 및 설치
```bash
./gradlew :app:assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 1.2 카메라 권한
| 항목 | 기대 결과 | Pass/Fail |
|---|---|---|
| 최초 실행 시 권한 다이얼로그 표시 | 카메라 권한 요청 팝업 | |
| 권한 허용 후 카메라 프리뷰 표시 | 후면 카메라 실시간 영상 | |
| 권한 거부 시 안내 메시지 표시 | "카메라 권한이 필요합니다" 텍스트 | |

### 1.3 AR 오버레이 기본 요소
| 항목 | 기대 결과 | Pass/Fail |
|---|---|---|
| 화면 중앙에 크로스헤어 표시 | 노란색 원+십자선 | |
| 하단에 힌트 카드 표시 | "객체를 터치하거나 중앙에 맞추세요" | |
| 화면 회전 없음 (portrait 고정) | 가로 회전해도 세로 유지 | |

## 크로스커팅 검증

### 메모리 누수 테스트
```
1. 앱 실행 → 1분간 카메라 이동 + 여러 객체 터치
2. Android Studio Profiler > Memory 탭
3. "Record" → 30초 대기 → GC 강제 실행
4. Heap Dump 분석: Bitmap, ImageProxy 누수 확인
```

| 항목 | 기대 결과 | Pass/Fail |
|---|---|---|
| Bitmap 누수 없음 | ImageProxy.close() 정상 호출 | |
| ViewModel 누수 없음 | Activity 재생성 후 메모리 해제 | |
| MediaPipe 리소스 해제 | onDispose에서 detector.close() 호출 | |

### 수명주기 테스트
| 항목 | 기대 결과 | Pass/Fail |
|---|---|---|
| 홈 버튼 → 복귀 | 카메라 재시작, 검출 정상 동작 | |
| 화면 회전 시도 | portrait 고정 유지 | |
| 다른 앱 전환 후 복귀 | 상태 복원, 크래시 없음 | |

### Logcat 에러 확인
```bash
adb logcat -s ObjectDetection CameraScreen VrtmvApplication | grep -i error
```
| 항목 | 기대 결과 | Pass/Fail |
|---|---|---|
| ObjectDetection 태그 에러 없음 | 정상 | |
| CameraScreen 태그 에러 없음 | 정상 | |
| ANR 없음 | 메인 스레드 블로킹 없음 | |

## 비교 실험 (논문용, 4단계 완료 후)

### 실험 설계
| 조건 | 입력 방식 | 측정 항목 |
|---|---|---|
| Baseline (전체 이미지) | 640x480 전체 프레임 → VLM | 지연시간, 메모리, 설명 품질 |
| Proposed (ROI 크롭) | 터치 선택 영역 크롭 → VLM | 지연시간, 메모리, 설명 품질 |

### 측정 지표
| 지표 | 측정 방법 |
|---|---|
| 추론 지연시간 (ms) | `System.currentTimeMillis()` 전후 차이 |
| 메모리 사용량 (MB) | `Runtime.getRuntime().totalMemory()` |
| 설명 관련성 | 사람 평가 (1-5점, 5명 이상) |
| 프레임 드롭률 | Profiler > Frame timeline |
